DIRSITE

Development aid intended to provide convenience by offering CLI input and output as browser-viewable web pages. When installed above a python package, prefab command lines are presented in tables. Other table contents may be assigned in a configuration file.

Installation

DIRSITE files may be copied into any directory to be used for executing arbitrary CLI instructions.

  • Commands are executed with the privileges of the server login.
  • URL may not require encoding. Example: http://localhost:9999/dirsite.py?cl=more dirsite_files\readme.html
  • URL may require encoding. Example for Windows: http://localhost:9999/dirsite.py?cl=dir && more dirsite_files\readme.html. The && operator may not be parsed when used in a URL. Improved results may be seen by using the encoded URL, http://localhost:9999/dirsite.py?cl=dir%20%26%26%20more%20dirsite_files\readme.html

Python Package Assistant

When configured, test by going to http://localhost:9999/dirsite.py?cl=.

DIRSITE Configuration File dirsite_files/dirsite.ini

[server]
port = 9999
increment_on_fail = False

[appearance]
navigate_by_post = False
reload_time = 60
first_page = dirsite.py?cl=
Some configuration changes to consider:
  • To see the interface with buttons rather than links, change navigate_by_post to True. When True, custom URLs can no longer be typed into the URL line.
  • To change the dirsite_cl.py refresh rate to once every 5 minutes, change reload_time to 300 (seconds). When navigate_by_post is True, reload is disabled to prevent the loss of persistent page view in the browser.
  • At server start, automatically load a view page by changing first_page value to http://localhost:9999/dirsite.py?cl=

Some configuration changes to consider:
  • To change the view shown when the command line is entered, change the [template] selection assignment.
  • To add a directory wide command... (Note: <n> indicates a number denoting order of row appearance, starting from 1.)
    • 'dir_header_<n>' for the table header of the command.
    • 'dir_content_<n>' for the command as it appears on command line.
    • If necessary, the following variables are available at runtime: dirtest.
  • To add a file specific command... (Note: <n> indicates a number denoting order of column appearance, starting from 1.)
    • 'file_header_<n>' for the table header of the command.
    • 'file_content_<n>' for the command as it appears on command line.
    • 'file_dependency_<n>' required by command as it appears on command line.
    • If necessary, the following variables are available at runtime: dirtest.
